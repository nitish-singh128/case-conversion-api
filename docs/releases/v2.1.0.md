## Release v2.1.0 — Architecture & DevOps Hardening

This release marks the completion of our "Clean Monorepo" initiative. We have restructured the project internals to improve maintainability and standardized our CI/CD pipelines for better observability.

### Monorepo Restructuring

- **Documentation Centralization:** Migrated all technical design documents, architectural diagrams, and performance reports to the `/docs` folder.
- **Improved Scannability:** Cleaned up the root directory to focus on core configuration and orchestration files.

### CI/CD Pipeline Standardization

- **Domain-Driven Naming:** Standardized all GitHub Action workflows to follow a strict naming convention:
  - `native-*`: For C++ Core Engine quality, security, and CI.
  - `dotnet-*`: For the API Gateway and P/Invoke integration layers.
  - `full-stack-*`: For end-to-end orchestration and matrix builds.
- **Badge Synchronization:** Updated README badges to reflect the new workflow hierarchy, ensuring real-time status visibility.

### Technical Debt & Quality

- **Security Audit:** Re-indexed CodeQL scanning to cover the renamed native security paths.
- **Quality Guard:** Updated Clang-Tidy and GTest workflows to ensure the native core remains 100% compliant with C++17 standards.
