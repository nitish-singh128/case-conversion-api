# Release v2.2.0 — The "Memory Sovereignty" Milestone

This major engineering update introduces deterministic resource management and significantly elevates the performance ceiling of the native conversion engine. By transitioning from manual memory handling to RAII and the Rule of Five, the system achieves industrial-grade stability alongside a substantial throughput improvement.

---

## High-Performance Engineering

- **Move Semantics Integration**  
  Implemented C++17 move constructors and assignment operators to eliminate redundant deep copies during the ABI hand-off.

- **Throughput Breakthrough**  
  Benchmarked at **2,827 req/s** on Apple M2 hardware — a **25% increase** over the v2.1.0 baseline.

- **Latency Optimization**  
  Achieved a **42% reduction in median (P50)** response times, now sustained at **22 ms** under heavy load.

---

## Memory Sovereignty & Safety

- **Zero-Leak Protocol**  
  Hardened the marshalling contract between .NET 8 and C++ to ensure **100% deterministic cleanup** of unmanaged buffers.

- **ABI Guard**  
  Reinforced the *"Callee-Allocates, Caller-Frees"* pattern to eliminate **double-delete risks** and prevent dangling pointers under high concurrency.

- **Saturation Reliability**  
  Validated through a **1,000,000-request endurance test** with **zero native memory growth**.

---

## DevOps & Distribution

- **ARM64 Native Builds**  
  CI/CD pipeline fully optimized for Apple Silicon (M-series), leveraging macOS runners for native compilation.

- **Stripped Production Binaries**  
  Release artifacts are stripped of debug symbols, reducing binary size while preserving runtime performance.

- **Self-Contained Artifacts**  
  Native `.dylib` is now bundled within the .NET publish directory, enabling **plug-and-play deployment**.
