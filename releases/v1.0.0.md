# Release v1.0.0: High-Performance Case Conversion Engine

## Overview

This release marks the stable deployment of the Word Case Conversion Engine, a polyglot system designed for high-throughput string processing. By bridging a Native C++17 Engine with a .NET 8 Managed Layer, we achieve near-hardware performance with the flexibility of a modern Web API.

## Technical Highlights

1. Apple Silicon Optimization (M2)

Hardware-Targeted Parallelism: Explicitly optimized for Apple M2 P-Cores (Performance Cores).

Vectorization: Leverages ARM64 SIMD instructions where applicable for batch string operations.

2. Polyglot Architecture

Zero-Copy Interop: Utilizes advanced P/Invoke patterns with a "Callee-Allocates, Caller-Frees" memory strategy to minimize overhead.

Hybrid Memory Management: Native memory is rigorously managed via C++ smart pointers and exposed through a thread-safe C-linkage layer.

3. Resilience & Observability

Soak Tested: Successfully completed a 200,000-request soak test with 100% success rate and stable memory consumption.

Distributed Tracing: Full integration with OpenTelemetry (OTEL), ensuring Trace ID propagation across the Native/Managed boundary.

## Artifacts

- WordCaseEngine-ARM64.zip: Contains the standalone .NET 8 API binaries and the libProcessStringDLL.dylib optimized for Apple Silicon.

- SourceCode-Metadata.zip: Includes database schemas and architectural documentation.

## Installation

1. Download and extract WordCaseEngine-ARM64.zip.

2. Ensure the .dylib is in the same directory as the executable.

3. Run the API:

```Bash
./DotNetAPI --urls "http://localhost:5000"
```
